equals() & hashCode(): Vertragsfallen

In Java muss der Vertrag zwischen equals() und hashCode() eingehalten werden: Wenn zwei Objekte gleich sind (equals() gibt true zurück), müssen sie den gleichen Hashcode besitzen. Andernfalls verhalten sich Hash-basierten Collections (z.B. HashSet, HashMap) unerwartet.
